## 2. 实践流程

1. **本地仓库初始化**: 在 `G:/机器学习` 目录下执行 `git init`，创建本地 Git 仓库。
2. **代码追踪与提交**: 使用 `git add .` 将所有实验代码和指导手册放入暂存区，并使用 `git commit` 生成版本快照。
3. **远程仓库关联**: 通过 `git remote add origin` 绑定 GitHub 公开仓库。
4. **代码推送**: 使用 `git push -u origin main` 将本地主分支代码推送到云端。
5. **身份配置**: 实践中途使用 `git config --global user.name` 和 `user.email` 重新配置了正确的提交者身份信息。

## 4. 遇到的问题及解决方法

- **问题1：远程地址输入错误导致找不到仓库 (Repository not found)**
  - **描述**: 在初次绑定远程仓库时，不小心在 url 结尾多敲了一个波浪号 `~`，导致 push 时报错。
  - **解决**: 使用 `git remote rm origin` 删除了错误的远程地址配置，随后重新使用 `git remote add origin` 绑定了干净、正确的 URL，成功解决。
- **问题2：推送时遇到 403 权限拒绝 (Permission denied)**
  - **描述**: push 时终端报错 403，提示 `ly6666323` 账号没有权限推送到当前 `ly20060420-wq` 的仓库，因为系统记住了旧账号。
  - **解决**: 打开 Windows 的“凭据管理器”，在“Windows 凭据”中找到了旧的 GitHub 登录记录并删除。再次执行 `git push` 时重新触发了浏览器授权弹窗，登录正确账号后推送成功。
- **问题3：命令行空格遗漏**
  - **描述**: 在终端手快敲成了 `git add.`，Git 提示找不到该命令。
  - **解决**: 查阅提示后，意识到 `add` 和 `.` 之间必须有空格，补上空格后执行 `git add .` 成功将文件加入暂存区。

## 5. Git学习心得

通过这次实践周的任务，我彻底告别了以前写代码靠“复制文件夹+重命名（比如：最终版、绝对不改版）”的原始备份方式。
在经历了 URL 拼写错误、403 账号权限冲突、以及本地用户名与 GitHub 账号不一致等真实的坑之后，我深刻理解了本地版本库（commit）和远程仓库（push）的区别。现在我已经能在本地从容地查看代码的修改历史，并且掌握了用 Git 来管理我的机器学习实验代码，这为后续参与更大的项目协作打下了很好的底子。
